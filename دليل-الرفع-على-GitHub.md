# كيف ترفع التطبيق على GitHub وتشغّله (خطوة بخطوة)

لديك الآن نسخة كاملة جديدة من التطبيق. اتبع هذه الخطوات لرفعها على GitHub وتشغيلها برابط حيّ مجاني.

---

## الطريقة الأسهل (عبر المتصفح — بدون أوامر)

### ١) أنشئ مستودعاً جديداً
1. ادخل **github.com** وسجّل الدخول (اسمك Jumbo84).
2. اضغط **+** (أعلى يمين) ← **New repository**.
3. الاسم: `alula-journey-time`
4. اختر **Public** (لتعمل GitHub Pages مجاناً) أو Private.
5. اضغط **Create repository**.

### ٢) ارفع الملفات
1. في صفحة المستودع الجديد، اضغط **uploading an existing file** (أو Add file ← Upload files).
2. فُكّ ضغط ملف `alula-journey-time.zip` على جهازك أولاً.
3. اسحب **كل الملفات والمجلدات** بداخله (index.html، app.js، manifest.json، sw.js، README.md، مجلد assets) وأفلتها في الصفحة.
4. اكتب وصفاً (مثلاً "initial version") واضغط **Commit changes**.

### ٣) فعّل GitHub Pages (الرابط الحيّ)
1. في المستودع، اذهب **Settings** ← **Pages** (القائمة اليسرى).
2. تحت **Build and deployment → Source**، اختر **Deploy from a branch**.
3. اختر الفرع **main** والمجلد **/ (root)**، ثم **Save**.
4. انتظر دقيقة، ثم حدّث الصفحة — سيظهر رابطك:
   `https://jumbo84.github.io/alula-journey-time/`

### ٤) افتحه على جوالك
- افتح الرابط **على جوالك** (الكاميرا والبوصلة تحتاج جهازاً حقيقياً + HTTPS، وGitHub Pages يوفّره).
- في AR View، اضغط **"Enable camera & compass"** للسماح بالأذونات.
- إن كنت بعيداً عن العلا، سيعمل **Demo Mode** تلقائياً وتظهر كل المعالم.

---

## ملاحظات مهمة

- **يعمل فوراً** بلا أي تثبيت أو بناء — مجرّد ملفات HTML/JS.
- **الخريطة والكاميرا** تعملان على الإنترنت الحقيقي (في بيئة الاختبار قد تُحجب مكتبة الخريطة، لكنها تعمل على GitHub).
- **لإضافة صور حقيقية:** ضع صورك في `assets/photos/` ولوجوهات الخدمات في `assets/logos/` (صورك أو من ويكيميديا — لا صور محمية).
- **للتحديث لاحقاً:** عدّل الملف على GitHub مباشرة، أو ارفع نسخة جديدة، واضغط Commit.

## ماذا يحتوي المشروع

| الملف | الوظيفة |
|------|---------|
| `index.html` | هيكل التطبيق + كل التنسيق |
| `app.js` | كل المنطق: التبويبات، AR، البوصلة، الخريطة، الكاميرا، اللغات |
| `manifest.json` | إعداد PWA (قابل للتثبيت على الجوال) |
| `sw.js` | العمل دون اتصال (تخزين مؤقت) |
| `assets/places.json` | البيانات الحقيقية (١٩ موقعاً + ١٤ خدمة) |
| `assets/icon-*.png` | أيقونات التطبيق |
| `README.md` | توثيق المشروع (يظهر في صفحة GitHub) |

---

## نصيحة أخيرة

بعد رفع المشروع، **رابط GitHub Pages هو ما تعرضه على الإدارة** — يفتحونه على أي جوال دون تثبيت، ويجرّبون التجربة مباشرة. هذا أقوى بكثير من شرح نظري.
